# 구현 단계 가이드 — 기침 소리 기반 화자 식별 시스템

> 작성일: 2026-07-06 · 연계: 개발 생명주기 7단계, UML 4종, 스토리보드·와이어프레임(S0~S4·M1)
> 확정 스택: 엣지 = 라즈베리파이 5(보유) + Python / 서버 = FastAPI / 프론트 = React + Vite / 식별 = 사전학습 임베딩(ECAPA-TDNN) + 코사인 매칭

## 0. 전체 로드맵 (8단계)

| 단계 | 내용 | 산출물 | 검증 |
|------|------|--------|------|
| P1 | 프로젝트 골격 + Git | 모노레포 구조 | 3개 모듈 실행 확인 |

## P1. 프로젝트 골격 (0.5일)

```
cough-id/
├── edge/                  # 라즈베리파이 5
│   ├── audio_capture.py   # AudioCapture (링버퍼)
│   ├── cough_detector.py  # CoughDetector (에너지+CNN)
│   ├── event_sender.py    # EventSender (재시도 큐)
│   ├── main.py
│   └── requirements.txt   # sounddevice, numpy, requests, tflite-runtime
├── server/
│   ├── app/
│   │   ├── main.py        # FastAPI 엔트리
│   │   ├── api/           # IngestAPI 라우터 (events, persons, alerts, auth)
│   │   ├── ml/            # FeatureExtractor, SpeakerIdentifier
│   │   ├── core/          # AlertEngine, WebSocket 매니저
│   │   ├── models.py      # Person, CoughEvent, Alert (SQLAlchemy)
│   │   └── db.py
│   └── requirements.txt   # fastapi, uvicorn, sqlalchemy, speechbrain, torchaudio
├── dashboard/             # React + Vite
└── docs/                  # 기존 UML·스토리보드 md 이동
```

- VS Code 멀티루트 워크스페이스(`cough-id.code-workspace`)로 3개 폴더 등록. 확장: Python, Pylance, ESLint, Prettier.
- Git 저장소 초기화 → GitHub 팀 리포. 브랜치 규칙: `main` 보호, `feat/edge-*`, `feat/server-*`, `feat/dash-*`로 팀원 분담.
- `dashboard/`는 `npm create vite@latest dashboard -- --template react`로 생성.

## P2. DB 스키마 + 서버 API 뼈대 (1~2일)

Class 다이어그램의 Person·CoughEvent·Alert를 SQLAlchemy 모델로 그대로 구현. DB는 개발 단계 SQLite → 필요 시 PostgreSQL 전환(SQLAlchemy라 코드 무수정).

| 엔드포인트 | 기능 | 연계 |
|-----------|------|------|
| POST /events | 엣지에서 wav+메타 수신 (multipart) | FR-03, Sequence |
| GET /events?from&to&person&unknown | 이력 조회·필터 | FR-06, S2 |
| GET/POST/DELETE /persons | 화자 CRUD·등록 | FR-09, S3 |
| POST /persons/{id}/samples | 등록용 샘플 업로드 | S3a 마법사 |
| GET/PUT /alerts, /alert-rules | 알림 이력·규칙 | FR-07, S4 |
| POST /auth/login | JWT 발급, 역할(admin/guardian) | NFR-06, S0 |
| WS /ws/feed | 실시간 이벤트 푸시 | NFR-03, S1 |

- 이 단계에서는 식별 모듈을 **스텁**(항상 unknown 반환)으로 두고 API 흐름만 완성.
- 검증: `uvicorn` 실행 → `/docs`(Swagger)에서 전 엔드포인트 수동 테스트 + pytest로 CRUD 자동화.

## P3. 식별 모듈 — ML 파이프라인 (3~5일, 최대 리스크 구간)

1. **FeatureExtractor**: torchaudio로 16kHz 리샘플 → log-mel 스펙트로그램 (`to_logmel()`).
2. **SpeakerIdentifier**: SpeechBrain 사전학습 ECAPA-TDNN(`speechbrain/spkrec-ecapa-voxceleb`)으로 임베딩 추출(`embed()`) → 등록 화자 임베딩과 코사인 유사도(`match()`), 임계치 미달 시 unknown(FR-05).
3. **화자 등록**: 샘플 N개 임베딩 평균을 `embedding_ref`로 저장(원본 음성 비보존 — NFR-06).
4. **오프라인 평가 스크립트**: 팀원 기침 녹음 각 10회 이상 수집 → 등록/테스트 분할 → 정확도·임계치 곡선 출력. **임계치는 감이 아니라 이 곡선으로 결정.**
5. 리스크 대비: 기침(1초 미만 비언어음)에서 정확도 부족 시 → 샘플 수 증가, 기침 데이터로 fine-tuning, 또는 기침+직후 숨소리 포함 3초 구간 사용 순으로 시도.

검증: 혼동 행렬 기준 팀원 4~5인 식별 정확도 목표치(생명주기 문서 기준) 달성 확인 후 P2 스텁 교체.

## P4. 엣지 — 기침 검출·전송 (2~3일)

**4a. PC에서 먼저 개발** (마이크만 있으면 됨):
- AudioCapture: sounddevice 스트림 → 링버퍼(수 초).
- CoughDetector: 1차 에너지 임계치(저비용 상시) → 2차 경량 CNN(YAMNet/tflite) 기침 판정 → 전후 여유 포함 2~3초 절단(TC-03).
- EventSender: POST /events, 실패 시 로컬 큐 저장·지수 백오프 재전송(TC-05).

**4b. RPi 5 포팅**:
- USB 마이크(또는 ReSpeaker 어레이 — DOA 쓸 경우) 연결, `arecord -l`로 장치 확인.
- VS Code **Remote-SSH**로 RPi에 직접 붙어 개발 — 로컬과 동일한 편집 경험.
- systemd 서비스 등록으로 부팅 시 자동 시작·크래시 재시작.
- 검증: 실제 기침 → 서버 DB에 이벤트 도착까지 확인, 네트워크 차단 후 복구 시 재전송 확인.

## P5. 알림 엔진 + 실시간 피드 (1~2일)

- AlertEngine: 이벤트 저장 직후 `evaluate()` — 슬라이딩 윈도(예: 1시간 10회) 초과 시 Alert 생성 + 웹훅 발송(Discord/Slack URL로 데모). 중복 발송 방지 쿨다운 포함(TC-04).
- WebSocket 매니저: 이벤트·알림 발생 시 `/ws/feed` 구독자에 브로드캐스트 → S1 피드 ≤3초 갱신(NFR-03).

## P6. React 대시보드 (4~6일, 서버와 병행 가능)

라이브러리: react-router, TanStack Query(API 캐시), Recharts(차트), 네이티브 WebSocket. 스타일은 Tailwind 권장.

| 순서 | 화면 | 핵심 구현 |
|------|------|-----------|
| 1 | S0 로그인 | JWT 저장, 역할 분기(admin→S1, guardian→S2 읽기전용) |
| 2 | 공통 레이아웃 | 사이드바 + 헤더(디바이스 상태 표시) — 와이어프레임 1440px 기준 |
| 3 | S1 대시보드 | KPI 카드 4개, 24h 막대차트, WebSocket 실시간 피드, 경고 배너 |
| 4 | S2 이력 | 필터 바, 라인차트, 테이블 + 오디오 재생, CSV 내보내기 |
| 5 | M1 모달 | 파형·스펙트로그램(wavesurfer.js), 유사도 게이지, 화자 수정 |
| 6 | S3·S3a | 화자 카드 + 3단계 등록 마법사(브라우저 MediaRecorder 녹음) |
| 7 | S4 알림 | 알림 이력, 규칙 편집 폼, 클릭 시 S2 필터 연동 |

- Figma 와이어프레임과 화면별 1:1 대조하며 진행. 팀 분담 시 화면 단위로 나누기 좋음.
- Vite 프록시(`/api` → `localhost:8000`)로 CORS 회피.

## P7. 통합 테스트 (2일)

- TC-01·02: 실제 기침 → 엣지 검출 → 식별 → 대시보드 표시 E2E (등록/미등록 각각).
- NFR-03: 기침~화면 표시 지연 측정 스크립트(타임스탬프 로깅) — ≤3초 확인.
- 장애 시나리오: 서버 다운 중 기침 → 엣지 큐 적재 → 복구 후 유실 없음.
- 멀티 화자: 팀원 전원 등록 후 교차 식별, 오식별 시 M1 "화자 수정" 동작 확인.

## P8. 데모 리허설

Sequence 다이어그램을 그대로 발표 대본으로: 기침 → 3초 내 피드 표시 → 10회 반복 → 경고 배너 + 웹훅 수신 → S2 추이 확인 → M1 오디오 재생. 데모용 시드 데이터 스크립트(과거 7일치 가짜 이벤트) 준비.
